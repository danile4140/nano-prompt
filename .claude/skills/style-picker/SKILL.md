# 绘本风格生成器

根据 `input.txt` 原著调性，生成 **4 个画风明显区隔**的备选视觉风格提示词，写入 `style_choices.json`。
用户从中选定 1 个后，将其 `test_scene_prompt` 内容写入 `style_settings.txt`，作为后续 char-scene-generator 和 storyboard-generator 的 `style_setting` 基调。

---

## 第一步 — 读取原文

读取 `input.txt`，快速提取：
1. **故事调性**：轻松/冒险/神秘/温暖/奇幻？
2. **目标受众**：6-10 岁儿童/青少年？
3. **核心视觉元素**：场景特征、色彩关键词、重要道具
4. **禁忌**：避免真人/写实摄影风（本项目硬性）

---

## 第二步 — 生成 4 个风格

**生成原则：**

- **风格间必须有明显区隔**——不同的线条感、色彩饱和度、媒介质感、时代感，一眼就能看出差异
- **每个风格都要符合原文调性**——不能给轻松童话选黑暗哥特风
- **适龄**：受众是儿童或青少年，风格需亲切可读、角色表情清晰
- **禁止真人/写实照片风格**
- **禁止扁平明快插画**（flat vector / 北欧扁平风 / 无渐变几何色块风）
- **禁止版画风格**（engraving / woodblock / crosshatch 交叉影线复古版画）
- **禁止剪纸拼贴风格**（paper cut collage / torn paper layering / paper collage）
- **禁止油画风格**（oil painting / realistic oil medium / 厚重油画肌理）
- **禁止油画棒风格**（oil pastel / 油画棒笔触 / crayon gras）
- **禁止在风格描述中出现任何具体角色名或场景名**——风格是画风基调，不是内容描述
- **禁止使用"圆润"、"rounded"、"chunky"等体型描述词**——会导致生图模型将角色渲染成胖体型，用"可爱"、"亲切"、"生动"、"自然比例"等词替代

风格选取**完全由原文推导**，不限定媒介类型。判断标准：
- 从原文的故事调性、时代背景、情绪节奏中提炼视觉气质
- 选出的风格需与原文气质自然契合，而不是从固定列表里挑一个
- 只要不在禁止列表里，任何有质感的视觉风格都可以考虑

**4 个风格的差异化维度参考（用于确保 4 个风格之间有明显区隔）：**

| 维度 | 可能的对比轴 |
|------|-------------|
| 线条 | 精细轮廓 vs 笔触自由 vs 晕染无轮廓 |
| 色彩 | 高饱和明快 vs 柔和淡雅 vs 低饱和暖调 |
| 媒介质感 | 水彩纸感 vs 彩铅/蜡笔粗粒 vs 厚涂颜料肌理 vs 数字渲染 |
| 时代感/地域感 | 现代手绘 vs 复古绘本 vs 特定文化插画传统 |

**测试场景选取原则：**
- 从原文中选一个包含主要角色、场景元素丰富的段落作为统一测试场景
- 4 个风格必须用完全相同的场景，只改变风格描述部分

**提示词撰写原则：**
- `test_scene_prompt` 只写"要什么"（场景描述 + 风格正向关键词），不写"禁止什么"
- 禁止项（禁止写实、禁止扁平插画等）由 skill 层面管控，不应出现在输出的提示词文本里

---

## 第三步 — 适龄自检

对生成的每个风格逐条判断是否适合第一步提取的目标受众年龄段。

**不通过条件（任一满足即需丢弃重选）：**
- 主色调为暗夜色、深黑、深蓝紫等低明度色调
- 强烈戏剧性明暗对比（chiaroscuro、从下方打光、强烈投影）
- 整体氛围阴暗、沉重、压迫感强
- 视觉风格成人化（学术博物志、史书史诗、写实纪录感）
- 人物面部或表情模糊难以辨读

**通过标准（须全部满足）：**
- 色彩明亮或温柔（适合儿童的高饱和或淡雅色调）
- 角色表情清晰可读
- 氛围温暖/欢快/奇幻/冒险（即使带悬念感，也是童趣式悬念，而非成人恐惧）

**处理方式：**
逐风格判断，输出一句话结论（通过 / 不通过·原因）。不通过的风格立即丢弃，替换为同样满足差异化要求且适龄的新风格，直到 4 个风格全部通过为止。

---

## 第四步 — 写入 style_choices.json

将通过自检的 4 个风格写入 `style_choices.json`（UTF-8 无 BOM，LF 换行），结构如下：

```json
{
  "project": "<项目名>",
  "test_scene_base": "<统一测试场景的纯场景描述，不含风格词>",
  "styles": [
    {
      "id": "A",
      "name": "<风格名称>",
      "description": "<2-3句话描述核心视觉特征>",
      "test_scene_prompt": "<完整生图提示词：场景描述 + 风格关键词（只写正向词，不写禁止项）>"
    },
    ...
  ]
}
```

**JSON 写入注意事项：**
- description 或 prompt 文本内若需引用词语，必须用 `「」` 书名号，**禁止**在 JSON 字符串值内使用 ASCII 双引号 `"`（会破坏 JSON 结构）
- 写完后用 `python3 -c "import json; json.load(open('style_choices.json', encoding='utf-8')); print('OK')"` 验证合法性，如有报错立即修复再继续

写完后：
1. 告知用户文件路径
2. 简要展示 4 个风格名称和一句话描述，方便用户快速判断
3. 请用户选定风格，并提示下一步：将选定风格的 `test_scene_prompt` 写入 `style_settings.txt`

---

## 编码要求

写入文件时必须用 UTF-8 无 BOM + LF 换行符，与项目规范一致。
